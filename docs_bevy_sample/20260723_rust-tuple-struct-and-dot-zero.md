# Rust 入門：タプル構造体と `.0`（名前なしフィールドの読み方）

日付: 2026-07-23

「`score.0` の `.0` って何？なんで急に『タプル』が出てくるの？」という素朴な疑問への回答。
`Score(pub usize)` のような書き方（タプル構造体）と、その中身を `.0` で触る仕組みを最小限で説明する。

この基礎を使っている実例は [[20260723_deref-newtype-vs-dot-zero]]（`**` をやめて `.0` に統一した話）を参照。

---

## 1. まず「タプル」とは

タプル = **複数の値を順番に並べただけ**の型。名前は付けず、丸カッコで囲む。

```rust
let t = (10, "hello", true);
```

中身の取り出しは **番号（位置）** で行う。番号は 0 から始まる。

```rust
t.0   // → 10       （0 番目）
t.1   // → "hello"  （1 番目）
t.2   // → true     （2 番目）
```

ポイント：**タプルの中身に名前は無い。あるのは位置番号だけ**。だから `.0`, `.1`, `.2` … で取り出す。

## 2. 構造体には「名前付き」と「名前なし」がある

### ① 名前付きフィールドの構造体（普通のやつ）

```rust
struct Score {
    value: usize,   // フィールドに value という名前がある
}

let s = Score { value: 0 };
s.value             // 名前でアクセス
```

### ② 名前なしフィールドの構造体 = タプル構造体

フィールドを **丸カッコで並べる** と、名前のない構造体になる。これを **タプル構造体（tuple struct）** と呼ぶ。

```rust
struct Score(usize);   // フィールドに名前がない

let s = Score(0);
s.0                 // 名前が無いので「番号」でアクセス
```

見た目のとおり「タプル `(usize,)` に `Score` という名前を付けたもの」というイメージ。
中身の取り出し方もタプルと同じで、**位置番号 `.0`, `.1` …** を使う。

## 3. だから `.0`

`Score(usize)` はフィールドが 1 個だけのタプル構造体。その 1 個は **0 番目**なので `.0` で触る。

```rust
struct Score(usize);
//           ~~~~~ これが 0 番目のフィールド

let mut s = Score(42);
s.0            // → 42
s.0 += 1;      // 0 番目のフィールドを +1 → Score(43)
```

フィールドが複数あれば `.0`, `.1` … と増える:

```rust
struct Point(f32, f32);   // 0 番目 x, 1 番目 y
let p = Point(1.0, 2.0);
p.0   // → 1.0
p.1   // → 2.0
```

## 4. このリポジトリでの使われ方

`Score` / `Lives` は「`usize` を 1 個だけ包んだ」タプル構造体（newtype とも呼ぶ）。中身は `.0`。

```rust
// game_engine/src/components.rs
#[derive(Resource, Deref, DerefMut)]
pub struct Score(pub usize);   // 中身の usize は .0

#[derive(Resource, Deref, DerefMut)]
pub struct Lives(pub usize);
```

```rust
score.0 += 1;                       // スコアを 1 増やす
lives.0 = lives.0.saturating_sub(1); // ライフを 1 減らす
```

`pub usize` の `pub` は「このフィールドを外のモジュールからも触ってよい」という公開設定。
名前が無いフィールドでも `pub` は付けられる。

## 5. なぜわざわざ 1 個だけ包む？（newtype）

`usize` をそのまま使わず `Score(usize)` / `Lives(usize)` と包むのは、**型で意味を区別する**ため。

- ただの `usize` だと、スコアもライフも同じ型で取り違えやすい。
- `Score` と `Lives` に分けておけば、コンパイラが「スコアが欲しい所にライフを渡した」等を弾ける。

Bevy では Resource を型で識別する（[[20260716_bevy-resource-res-resmut-basics]]）ので、
「1 個だけ包んだタプル構造体」は Resource を作るときの定番パターンになっている。

## 6. まとめ

- **タプル** = 名前なしで値を並べた型。中身は番号 `.0`, `.1` … で取り出す。
- **タプル構造体** = フィールドを丸カッコで並べた、名前なしフィールドの構造体（例: `Score(usize)`）。中身は同じく `.0`, `.1` …。
- `Score(pub usize)` はフィールド 1 個 → 中身は **0 番目 = `.0`**。
- `usize` を `Score` / `Lives` で包むのは、型で意味を区別するため（newtype）。
